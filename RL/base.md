# 基础概念
- **Interleaved Reasoning and Actions**
简单来说，这是一种 **“边想边做，边看边修”** 的循环模式 。在传统的 AI 模式下，AI 往往是“一气呵成”地给出一大段话。但在 Kimi K2.5 中，它会像人类解决难题一样：
想（Reasoning）： 思考下一步该干什么（例如：“我需要找到这个迷宫的出口”）。
做（Action）： 调用工具去执行（例如：运行一个搜索或一段 Python 代码） 。
看（Observation）： 观察执行的结果（例如：代码报错了，或者找到了一个死胡同）。
再想： 根据反馈修正思路，进行下一步。
这种“推理 -> 行动 -> 结果反馈 -> 再推理”的无限循环，就是“交替（Interleaved）”的含义 。
